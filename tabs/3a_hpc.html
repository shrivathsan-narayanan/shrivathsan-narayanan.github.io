GPU Parallelisation of Rotor Induced Inflow Computation

The free-vortex wake method (FVM) model developed for numerical rotorcraft aerodynamics has been optimised with parallel-computing techniques using CUDA FORTRAN. The profiling of the solver carried out using GProf in Python, revealed that a typical single core simulation of a full-scale single rotor hover solution spent nearly 77% of its time in the Biot-Savart induced inflow computations. The routine is an O(N2) operation, and can be directly parallelised since the calculation of induced velocity by a given vortex filament on the other filaments present in the computational domain is independent for every filament in the domain, at a given instant of time.

The identified bottleneck in the Biot-Savart routine of the FVM solver has been optimised using CUDA FORTRAN by specifying the number of threads and blocks for the computation, depending on the total number of sources and targets present at each time step. A full-scale single rotor hover solution has been demonstrated to achieve at least 2-3× performance gains when simulated on a AMD Ryzen 7 5800H CPU with a 4GB NVidia GeForce RTX 3050 GPU. As per Amdahl's law, the maximum achievable performance gain is limited to ≈ 4.4× for a given problem size, which is largely constant for a given set of simulations.

Further optimisation of other potential routines is currently underway to improve the solver performance across larger problem sizes, to achieve Gustafson's speedup.
